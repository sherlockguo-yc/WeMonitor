# 替换（或引入）前端 JS/CSS 库前，必须在本地先跑一次

## 规则

替换 Chart.js → uPlot、jQuery → React、Bootstrap → Tailwind 等**前端库**之前，**必须**先在本地浏览器打开页面验证基本渲染，再提交代码。

## 流程

1. `npm start` 或 `node server.js` 启动本地服务
2. 浏览器打开对应页面，确认图表/组件能**正常渲染**（非空白、非崩溃、非布局错误）
3. 检查 Console 无报错（红色 error）
4. 切换多个数据范围（1h / 6h / 24h / 7d）确认不同数据量下都能正常显示
5. 以上全部通过后，再 `git commit && git push`

## 禁止

- ❌ 下载 CDN 文件后直接 `git push`，不打开浏览器看一眼
- ❌ 只验证 JS 语法（`node -e "new Function(...)"`）就认为没问题——CSS 布局问题（如 uPlot `min-content` 塌陷）语法检查发现不了
- ❌ 先推进性的功能（tooltip、cursor、配色），再回来修「渲染空白」——必须先保证图表能**出现**，再说好不好看

## 背景

2026-07-19 替换 Chart.js → uPlot 时，连续 4 次部署后才发现问题：
1. uPlot 默认 `width: min-content` 导致图表塌陷 0px（纯 CSS 问题，语法检查通过）
2. uPlot time scale 用 Unix 秒而非毫秒（API 单位错误，语法检查通过）
3. `cursor.focus` 缺少 DOM 容器报错 `Cannot read properties of undefined`
4. tooltip 坐标系混合视口/局部坐标，偏移几百 px

这 4 个问题如果在推送前本地浏览器打开一次，可以在 1 轮内全部发现，实际浪费了 5+ 轮交互。
