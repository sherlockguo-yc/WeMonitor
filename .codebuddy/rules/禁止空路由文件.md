# 禁止提交空的实现文件（路由 / 模块）

## 规则

- 禁止提交「空的实现文件」——即文件存在但内容为空（0 字节）或只有注释/`require` 占位、没有实际 `module.exports` 的路由 / 模块文件。
- `server.js` 启动时必须校验 `require()` 的路由模块是否为有效的 express Router（`typeof === 'function'`），不是则启动失败并给出清晰错误，而非等到 `app.use()` 才抛 `TypeError: app.use() requires a middleware function`。

## 背景

WeMonitor 的 `routes/deploy.js` 是空文件（0 字节），`require()` 返回 `{}` 而非 Router。`server.js` 的 `app.use('/api/v1/deploy', require('./routes/deploy'))` 在启动即抛 `TypeError`，导致整个服务崩溃、用户无法访问。该文件是开发中途残留的占位文件。

## 正确做法

- 占位 / 待开发路由请在 `server.js` 中注释掉 `app.use` 行，并标注 `// TODO: 待开发完成再启用`，而非保留空文件被 `require`。
- 启动脚本中增加校验：遍历所有 `require` 的路由模块，确认是函数后再 `app.use`。

## 不要忘记

这是项目级约束，每次新增 / 修改路由文件时**自动检查**，不需要用户提醒。
