# N150 服务部署后验证清单

新功能代码 push → CI 构建 → N150 auto-deploy 完成后,**必须**执行以下验证,确保功能在生产环境可用。

## 验证步骤

### 1. 确认版本已部署

```bash
ssh weweb "cat ~/wemonitor/.version"
```

版本应与最新 push 的 commit sha 一致。

### 2. 确认 server 进程已重启

```bash
ssh weweb "lsof -ti:18990"   # PID 应为新的
```

### 3. 确认关键环境变量

```bash
# 检查 server 进程环境中是否有必要变量
ssh weweb "cat /proc/\$(lsof -ti:18990)/environ | tr '\0' '\n' | grep GITHUB_TOKEN"
```

### 4. 端到端验证新功能

- 用浏览器访问 `wemonitor.sherlockguo.com/<new-page>`,确认页面渲染正常
- 打开 DevTools Network 面板,确认 API 调用返回 200(非 302/403/404)
- 如果新功能依赖外部 API(如 GitHub),确认调用未被限流(403)

### 5. 确认 cron 日志正常

```bash
ssh weweb "tail -5 /tmp/wemonitor-update.log"
```

最后一次日志不应有错误,下次 cron 触发间隔正常(≤ 2 分钟)。

## 红灯信号

以下任一出现则说明部署未生效或有运行时故障：

- `.version` 与 commit sha 不一致
- API 返回 403(Rate Limit)、404(路由未挂载)、302(鉴权中间件 bug)
- cron 日志出现 "API 返回非预期状态码"、"版本校验失败"
- 页面白屏/加载失败

**上述任一出现,禁止宣告「部署完成」,必须先诊断修复。**
