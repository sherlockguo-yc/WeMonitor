# WeMonitor — 数据导出

> 状态：规划中

## 1. 背景

当前指标数据只能在 Dashboard 图表中查看，无法导出供外部分析或在其他工具中使用。

## 2. 功能需求

### 2.1 导出格式

支持两种格式，用户选择：

| 格式 | 适用场景 |
|------|---------|
| CSV | Excel / Google Sheets 分析、Grafana 数据源 |
| JSON | 程序化分析、备份、迁移 |

### 2.2 导出维度

| 维度 | 选项 |
|------|------|
| 时间范围 | 最近 1 小时 / 6 小时 / 24 小时 / 7 天 / 自定义起止时间 |
| 服务 | system / WeMusic / 全部 |
| 指标 | CPU / 内存 / 磁盘 / 网络 / 全部 |

### 2.3 导出粒度

- 原始粒度（1 分钟）—— 精细分析，数据量大
- 聚合粒度（1 小时）—— 趋势分析，数据量小

选择的时间范围 > 限制自动切换粒度（如 > 7 天强制 1 小时粒度，防止文件过大）。

### 2.4 数据内容

**CSV 格式**：
```
timestamp,service,metric_name,labels,value
1711555200000,system,cpu_usage_percent,{},45.2
```

**JSON 格式**：
```json
[
  { "timestamp": 1711555200000, "service": "system", "metric_name": "cpu_usage_percent", "labels": {}, "value": 45.2 }
]
```

### 2.5 文件名

`wemonitor-export-{service}-{from}-{to}.csv`

## 3. 技术方案

### 3.1 后台导出

前端选择参数 → API 生成文件 → 返回下载链接或直接 response blob。

**直接响应**（数据量小时）：设置 `Content-Disposition: attachment`，流式写入 response。

**异步导出 + 下载链接**（数据量大时）：
1. POST 提交导出请求 → 返回任务 ID
2. 后台生成文件到 `data/exports/` 目录
3. 前端轮询任务状态，完成后提供下载链接
4. `data/exports/` 定期清理（24 小时）

MVP 阶段用直接响应即可（大多数指标数据量在数万行以内）。

### 3.2 性能保护

- 限制单次导出最大行数（如 100 万行）
- 超过限制时提示缩小时间范围
- 使用流式写入，不将全部数据加载到内存

## 4. 前端交互

在「系统资源」页面的图表区域新增「导出」按钮：
- 下拉选择格式（CSV / JSON）
- 弹窗确认时间范围和粒度
- 点击「导出」→ 下载文件

## 5. API 端点

| 方法 | 端点 | 说明 |
|------|------|------|
| `POST` | `/api/v1/export/metrics` | 导出指标数据（body: service, metric_names[], from, to, format, granularity） |
