# WeMonitor — 告警规则与通知

> 状态：规划中

## 1. 背景

当前 WeMonitor 监控了 CPU/内存/磁盘/网络/服务健康等指标，但都是被动查看——用户必须打开 Dashboard 才能发现异常。加上告警能力才能形成完整的监控闭环。

## 2. 功能需求

### 2.1 告警规则

支持创建基于指标阈值的告警规则：

| 字段 | 类型 | 说明 |
|------|------|------|
| 名称 | string | 如「CPU 过高」 |
| 指标 | select | 从已有指标中选取（cpu_usage_percent / mem_usage_percent / disk_usage_percent 等） |
| 服务 | select | system / 具体服务 |
| 条件 | select | `>` / `<` / `==` |
| 阈值 | number | 如 90（表示 90%） |
| 持续时间 | number | 连续 N 分钟满足条件才触发（防止瞬时抖动） |
| 通知渠道 | multi-select | 企微机器人 / 页面内提醒 |
| 静默时段 | time range | 如 23:00-07:00 不告警 |
| 启用 | toggle | 开关 |

### 2.2 预设规则模板

创建规则页提供常见模板一键启用：
- CPU 使用率 > 90%，持续 5 分钟
- 内存使用率 > 85%，持续 5 分钟
- 磁盘使用率 > 90%
- 服务状态变为 unhealthy
- 部署失败

### 2.3 通知渠道

**企微机器人 Webhook**：
- 全局配置：Webhook URL（在设置页填写）
- 告警消息格式：`【WeMonitor 告警】CPU 使用率 92%，超过阈值 90%（N150，持续 5 分钟）`
- 恢复通知：指标回落到阈值以下时，发送「已恢复」消息

**页面内提醒**：
- 导航栏图标红点 + 数字角标
- 下拉面板展示最近告警列表

### 2.4 告警历史

- 告警触发记录列表（时间/规则/触发值/状态）
- 状态：`firing`（告警中）/ `resolved`（已恢复）
- 聚合展示：同一规则连续触发时不重复记录，合并为时间段

## 3. 技术方案

### 3.1 评估引擎

- `lib/alerting/evaluator.js` — 每次数据采集后运行，遍历所有启用的规则
- 查询最近 N 分钟的指标数据（N = 规则配置的持续时间）
- 所有数据点都满足条件 → 触发告警
- 已触发 + 当前不满足 → 发送恢复通知

### 3.2 去重

- 内存中维护 `{ ruleId → { firing: bool, since: timestamp } }` 状态
- 已在 firing 状态 → 不重复发通知
- 状态从 firing → normal → 发恢复通知

### 3.3 通知发送

- `lib/alerting/notifiers/` — 插件化通知渠道
- 企微机器人：POST JSON → Webhook URL
- 页面内提醒：写 SQLite `alerts` 表，前端轮询

## 4. 数据模型

```sql
CREATE TABLE alert_rules (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  service TEXT NOT NULL,
  metric_name TEXT NOT NULL,
  condition TEXT NOT NULL,        -- > / < / ==
  threshold REAL NOT NULL,
  duration_minutes INTEGER DEFAULT 5,
  channels TEXT DEFAULT '["web"]',  -- JSON: ["web", "wecom"]
  quiet_start TEXT,               -- "23:00"
  quiet_end TEXT,                 -- "07:00"
  enabled INTEGER DEFAULT 1
);

CREATE TABLE alert_history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  rule_id INTEGER REFERENCES alert_rules(id),
  trigger_value REAL NOT NULL,
  status TEXT NOT NULL,           -- firing / resolved
  fired_at INTEGER NOT NULL,
  resolved_at INTEGER
);
```

## 5. API 端点

| 方法 | 端点 | 说明 |
|------|------|------|
| `GET` | `/api/v1/alerts/rules` | 规则列表 |
| `POST` | `/api/v1/alerts/rules` | 创建规则 |
| `PUT` | `/api/v1/alerts/rules/:id` | 编辑规则 |
| `DELETE` | `/api/v1/alerts/rules/:id` | 删除规则 |
| `POST` | `/api/v1/alerts/rules/:id/toggle` | 启停规则 |
| `GET` | `/api/v1/alerts/history` | 告警历史 |
| `GET` | `/api/v1/alerts/active` | 当前活跃告警 |
| `PUT` | `/api/v1/alerts/settings` | 通知渠道配置（Webhook URL） |
