# WeMonitor — 深色模式

> 状态：规划中

## 1. 背景

WeMonitor 的 CSS 变量体系已有基础，实现深色模式只需定义深色主题的变量值并增加切换逻辑。

## 2. 功能需求

### 2.1 主题切换

- 侧栏底部「主题」切换按钮：浅色 / 深色 / 跟随系统
- 切换即时生效，无需刷新页面
- 偏好持久化到 localStorage

### 2.2 三种模式

| 模式 | 说明 |
|------|------|
| 浅色 | 当前默认效果 |
| 深色 | 深色背景 + 浅色文字 |
| 跟随系统 | 根据 `prefers-color-scheme` 自动切换 |

### 2.3 初始化逻辑

```
页面加载
  → 检查 localStorage 偏好
    → 有：「浅色」/「深色」→ 应用
    → 无或「跟随系统」→ 检查 prefers-color-scheme → 浅色/深色
```

监听 `matchMedia('(prefers-color-scheme: dark)')` change 事件，自动跟随。

### 2.4 过渡动画

切换主题时添加短暂 transition（0.3s），避免闪烁。

## 3. 技术方案

### 3.1 CSS 变量切换

当前 CSS 变量在浅色模式下定义：

```css
:root {
  --bg: #f5f5f5;
  --bg-card: #ffffff;
  --text: #333333;
  --text-dim: #888888;
  --border: #e0e0e0;
  --accent: #4a90d9;
}
```

深色模式通过 class 覆盖：

```css
body.theme-dark {
  --bg: #1a1a2e;
  --bg-card: #16213e;
  --text: #e0e0e0;
  --text-dim: #888888;
  --border: #2a2a4a;
  --accent: #5fa3e0;
}
```

### 3.2 图表适配

uPlot 图表的颜色通过 JS 传入，需要根据当前主题动态计算颜色值。

`common.js` 暴露 `getTheme()` 函数，各页面的图表初始化代码检查主题并使用对应颜色。

### 3.3 需要检查的视觉元素

| 元素 | 浅色 | 深色 |
|------|------|------|
| 页面背景 | `--bg` | ✅ 自动 |
| 卡片 | `--bg-card` | ✅ 自动 |
| 表格 | 斑马纹 | 调整对比度 |
| 模态框 | overlay + 白色卡片 | overlay 加深，卡片深色 |
| 表单 input | 白底黑字 | 深底浅字，边框可见 |
| 图表 | 浅色画布 | 深色画布 + 调整线条色 |
| Toggle switch | 灰底色 | 深灰底色 |
| 侧栏 | 略深底色 | 更深底色 |
| 状态颜色 | 绿/红/黄 | 保持饱和度稍降 |

### 3.4 FOUC 防护

在 `<head>` 中内联一段 JS，在页面渲染前读取 localStorage 并设置 `body.className`，防止浅色 → 深色的闪烁。

```html
<script>
  (function() {
    const theme = localStorage.getItem('theme');
    if (theme === 'dark') document.documentElement.classList.add('theme-dark');
    else if (!theme || theme === 'system') {
      if (matchMedia('(prefers-color-scheme: dark)').matches) {
        document.documentElement.classList.add('theme-dark');
      }
    }
  })();
</script>
```

## 5. 影响范围

**风险等级：低**。CSS 变量体系已完整，只需新增变量值和少量 JS。

- `public/css/style.css` — 新增深色变量值
- `public/js/common.js` — 新增主题切换逻辑 + 图表颜色适配
- `views/layout.ejs` — 新增 FOUC 防护脚本 + 主题切换按钮
- 各页面 JS 文件 — 图表初始化时使用主题感知颜色
