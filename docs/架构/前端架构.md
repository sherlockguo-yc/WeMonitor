# WeMonitor — 前端架构

> 状态：已实现

## 1. 概述

WeMonitor 采用 **SSR + SPA 混合架构**：
- **页面框架**：EJS 服务端渲染（layout + 侧栏导航 + 页面内容）
- **数据加载**：前端原生 `fetch()` + 局部刷新
- **图表**：uPlot（time series）+ React Flow（拓扑编辑器）

## 2. 模板体系

### 2.1 Layout

`views/layout.ejs` — 全局布局模板（`express-ejs-layouts`）：

```
┌──────────┬──────────────────────────────┐
│  侧栏导航  │         内容区               │
│  (250px)  │   <%- body %>               │
│           │                              │
│  版本信息  │                              │
└──────────┴──────────────────────────────┘
```

侧栏导航分三组（可折叠/展开）：
- **系统管理**：概览、系统资源、定时任务
- **服务管理**：服务状态、服务管理、CI/CD、备份
- **网络管理**：防火墙、Tunnel、网络拓扑

底部显示部署版本号和时间戳（从 `.version` 文件读取，注入到 `res.locals`）。

## 3. 前端基础设施

### 3.1 API 请求封装 (`common.js`)

`api(path)` — 统一请求封装：

- 自动注入 Session Cookie
- **前端缓存**：内存 Map，不同端点不同 TTL（15s ~ 120s）
- 401 → 自动跳转登录
- 网络错误 → 回退到过期缓存（优雅降级）
- 慢查询日志：> 200ms 打印警告

### 3.2 图表封装 (`common.js`)

`createTimeChart(containerId, datasets, yAxisOpts)` — uPlot 时间序列图封装：

- 自动处理 Unix 毫秒时间戳
- 自适应 Y 轴格式
- 统一颜色方案
- 统一 Tooltip 样式

### 3.3 工具函数 (`common.js`)

| 函数 | 用途 |
|------|------|
| `formatBytes` | 字节转可读格式 |
| `formatPercent` | 百分比格式化 |
| `formatTime` | 时间戳格式化 |
| `formatDateTime` | 日期时间格式化 |
| `formatUptime` | 运行时间格式化 |
| `escapeHtml` | XSS 防护 |

### 3.4 静态资源策略

| 资源 | 缓存时间 | 刷新机制 |
|------|---------|---------|
| `*.js` / `*.css` | 7 天 | `?v=<git-hash>` 版本号 |
| HTML 页面 | 不缓存 | EJS 动态渲染 |
| API 响应 | 不缓存 | 实时数据 |

### 3.5 本地依赖

Chart.js 和 Lucide 图标本地部署在 `public/` 目录，不依赖外部 CDN：
- `public/chart.js` → Chart.js
- `public/lucide.js` → Lucide icons
- 在 EJS layout 中通过 `<script>` 引入

## 4. 拓扑画布

详见 [网络拓扑](../功能规格/网络拓扑.md)。

### 架构

单一 React Flow 画布同时承担查看和编辑职责，通过 `readOnly` 状态切换：

```
/network 加载 → EJS 嵌入 editor.js → mountTopologyEditor()
  → App 渲染 React Flow 画布（默认 readOnly=true）
  → 只读：panOnScroll / 悬停 tooltip / 无拖拽
  → 点击「编辑」→ readOnly=false → Handle 出现 / 拖拽解锁 / 面板显示
```

不再有独立的静态 SVG 渲染层。

### 技术选型

React + @xyflow/react + Vite lib 模式（IIFE），嵌入 EJS 模板。

### 构建流水线

```
CI: npm ci → npm run build (Vite) → 输出到 public/topology-editor/
          editor.js (~243KB gzipped) + wemonitor.css
          ↓
N150: Express 静态服务 public/ → /topology-editor/editor.js
          ↓
页面: /network → EJS 加载 editor.js → 自动 mount
```

### 文件结构

```
topology-editor/
  src/
    main-embed.jsx       # IIFE 入口 → window.mountTopologyEditor()
    App.jsx              # 主组件：readOnly 切换 / 画布 / tooltip / 面板 / 脏状态追踪
    PropertyModal.jsx    # 节点/连线属性弹窗
    VersionModal.jsx     # 版本历史弹窗（列表 / 预览 / 恢复）
    api.js               # API 封装（含版本管理）
    nodes/
      TopologyNode.jsx   # React.memo 节点组件（Handle 在只读时隐藏）
```

### 特殊配置

- `define: { 'process.env': '{}' }` — IIFE 无 Node.js process.env
- `base: '/topology-editor/'` — CDN 路径
- `fileName: () => 'editor.js'` — 固定名 + `?v=<%= version %>` 破缓存

## 5. 前端开发规范

### 5.1 CSS 体系

- 纯 CSS，CSS 变量体系（`--bg` / `--text` / `--accent` / `--border` 等）
- 主题色通过 CSS 变量切换，所有颜色引用使用语义变量
- 字号基于 `--font-size` 基准值，使用 `calc()` 缩放

### 5.2 图标

- 统一使用 Lucide 图标库
- `currentColor` 继承文字颜色
- 按钮内图标：`display: inline-flex; align-items: center; gap: 5px`

### 5.3 UI 控件

- 二元状态（启用/禁用）→ Toggle Switch 开关
- 多选项 → 选项列表/勾选框组
- 确认操作 → 二次确认弹窗
