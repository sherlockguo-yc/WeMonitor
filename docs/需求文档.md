# WeMonitor 需求文档

> 2026-07-18 · N150 小主机服务器监控系统 · **已实现**

---

## 一、项目概述

WeMonitor 是一个轻量级 Node.js 监控系统，运行在 N150 小主机上，监控系统资源、服务健康状态和自定义应用指标，通过 Web Dashboard 可视化展示，支持 UFW 防火墙管理、Cloudflare Tunnel 管理、用户权限管理，通过 Cloudflare Tunnel 提供域名访问。

### 核心原则

- 单进程运行，内存占用 < 200MB，无外部组件依赖
- EJS 服务端渲染，Chart.js / Lucide 本地部署（不依赖外部 CDN）
- 全自动部署（git push → GitHub Actions → N150 cron → 自动更新）
- 静态资源浏览器缓存 7 天 + 版本号自动刷新（`?v=<hash>`）
- Session 持久化到 SQLite，服务重启后保持登录状态

## 二、用户认证与权限

### 2.1 认证

- 注册 / 登录 / 退出
- 密码 bcrypt 加密存储
- Session 基于 SQLite 持久化（`better-sqlite3-session-store`），7 天有效期
- 服务重启不影响登录状态

### 2.2 角色与权限

| 角色 | 说明 | 权限 |
|------|------|------|
| `admin` | 超级管理员 | 全部页面 + 用户管理 + 防火墙管理 |
| `user`（待审批） | 注册后默认状态 | 仅看到「等待审批」页面 |
| `user`（已激活） | admin 审批通过后 | Dashboard 全部查看权限（概览/系统/服务） |

### 2.3 授权流程

```
注册 → 首个用户自动 admin + active
注册 → 后续用户默认 pending → admin 在「用户管理」审批 → active
```

## 三、监控能力

### 3.1 系统指标（自动采集）

| 类别 | 指标 | 频率 |
|------|------|------|
| CPU | 使用率 | 30s |
| 内存 | 总量 / 已用 / 使用率 | 30s |
| 磁盘 | 各分区总量 / 已用 / 使用率 | 60s |
| 网络 | 入站 / 出站流量 | 60s |
| 其他 | 系统负载、CPU 温度、运行时间 | 60s |

### 3.2 服务健康检查

- TCP 端口探测（如 `127.0.0.1:5174` = WeMusic）
- 状态：`healthy` / `degraded` / `unhealthy`

### 3.3 应用指标

| 模式 | 触发方 | 接口 |
|------|--------|------|
| **Pull** | WeMonitor → WeMusic | 定时 scrape `GET /metrics` |
| **Push** | 外部服务 → WeMonitor | `POST /api/v1/metrics`（批量 JSON） |

## 四、Web Dashboard

后台管理风格：左侧导航菜单 + 右侧内容区。

| 页面 | 内容 | 权限 |
|------|------|------|
| **概览** | 系统资源实时卡片 + 趋势图（1h/6h/24h/7d）+ 服务健康 | 已激活用户 |
| **系统资源** | CPU / 内存 / 磁盘 / 网络趋势图，时间范围可选 | 已激活用户 |
| **服务状态** | 服务列表 + 健康检查历史 | 已激活用户 |
| **服务管理** | 注册/编辑/启用/禁用/删除被监控服务 | 已激活用户 |
| **防火墙** | UFW 规则查看、添加、删除 | 已激活用户 |
| **Tunnel** | Cloudflare Tunnel 状态、日志、远程重启 | 已激活用户 |
| **用户管理** | 用户列表、审批、删除 | 仅 admin |

## 五、UFW 防火墙管理

### 5.1 功能

- 表格列出所有 UFW 规则（端口、协议、动作、来源）
- 添加规则：端口 + 协议（TCP/UDP）+ 描述
- 删除规则：按编号删除，二次确认

### 5.2 技术实现

- Node.js 通过 `child_process` 执行 `sudo ufw` 命令
- N150 配置 sudoers 免密：`sherlockguo ALL=(ALL) NOPASSWD: /usr/sbin/ufw`

## 七、部署架构

### 7.1 自动更新流水线

```
git push master
    ↓
GitHub Actions（ubuntu-latest）
    npm ci → 打包 wemonitor.tar.gz → GitHub Release（tag=latest）
    ↓
N150 cron 每分钟运行 ~/wemonitor-update.sh
    查询 Release → 比对版本 → 下载（8 次重试）→ 解压 → restart
    ↓
N150 ~/wemonitor/
    ├── server.js / lib/ / routes/ / views/ / public/
    ├── node_modules/（CI 预编译）
    ├── data/（SQLite + sessions，rsync --exclude 保护）
    └── .version
```

### 7.2 网络访问

| 方式 | 地址 | 说明 |
|------|------|------|
| 内网 | `http://192.168.31.102:18990` | UFW 已放行 |
| 公网域名 | `https://wemonitor.sherlockguo.com` | CF Tunnel → :18990 |

### 7.3 Cache 策略

| 资源类型 | 缓存时间 | 刷新机制 |
|----------|---------|---------|
| `*.js` / `*.css` | 7 天 | `?v=<git-hash>` 自动更新 |
| HTML 页面 | 不缓存 | EJS 动态渲染 |
| API 响应 | 不缓存 | 实时数据 |
