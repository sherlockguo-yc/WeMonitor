# WeMonitor — 数据备份管理

> 状态：已实现

## 1. 功能概述

通过「数据备份」页面（`/backup`）管理各服务的 Cloudflare R2 备份状态，支持查看备份历史信息、开关备份功能。

## 2. 备份存储

- 平台：Cloudflare R2（S3 兼容对象存储）
- 存储桶：`n150-backups`
- 备份对象：各服务的数据文件（如 SQLite 数据库）

## 3. 备份配置

### 3.1 数据模型

```sql
CREATE TABLE backup_config (
  service_name TEXT PRIMARY KEY REFERENCES services(name),
  enabled INTEGER DEFAULT 0,
  last_backup_at INTEGER,
  last_backup_size INTEGER,
  r2_path TEXT
);
```

### 3.2 配置字段

| 字段 | 说明 |
|------|------|
| `service_name` | 关联的服务名 |
| `enabled` | 是否启用备份 |
| `last_backup_at` | 上次备份时间（Unix 毫秒） |
| `last_backup_size` | 上次备份大小（字节） |
| `r2_path` | R2 存储路径 |

## 4. 前端页面

### 4.1 备份列表

表格展示所有已注册服务及其备份状态：
- 服务名称
- 备份开关（Toggle）
- 上次备份时间
- 上次备份大小
- R2 路径

### 4.2 开关备份

Toggle 开关调用 `PATCH /api/v1/backup/services/:name/toggle`，即时生效。

## 5. API 端点

| 方法 | 端点 | 说明 |
|------|------|------|
| `GET` | `/api/v1/backup/services` | 备份服务列表 |
| `PATCH` | `/api/v1/backup/services/:name/toggle` | 开关备份 |
