# WeMonitor — 网络拓扑

> 状态：已实现

## 1. 功能概述

通过网络拓扑页面（`/network`）可视化展示 N150 家庭网络环境，包括从 Internet 到 N150 上各服务的完整链路。支持查看模式（静态 SVG + 动态状态）和编辑模式（React Flow 可视化编辑器）。

## 2. 拓扑结构

### 2.1 节点类型

| 节点类型 | 说明 |
|---------|------|
| Internet | 互联网 |
| CDN | Cloudflare CDN |
| ISP | 运营商 |
| 光猫 | 192.168.1.1 |
| 路由器 | 192.168.31.1（小米路由器） |
| 防火墙 | UFW |
| 服务器 | N150 小主机 |
| Tunnel | Cloudflare Tunnel |
| 服务 | WeMusic / WeMonitor / WeDownload 等 |

### 2.2 连线表示

- Internet → CDN → ISP → 光猫 → 路由器 → UFW → N150 → 各服务
- Cloudflare Tunnel 作为旁路连接（公网域名 → Tunnel → 本地服务）

## 3. 动态状态覆盖

拓扑图中的节点颜色根据实际状态动态变化：

| 状态来源 | 检测方式 | 影响节点 |
|---------|---------|---------|
| 物理拓扑 | Ping（光猫 192.168.1.1、路由器 192.168.31.1） | 光猫、路由器 |
| 路由器 API | 小米路由器 API（stok token 认证） | 路由器详情（型号/CPU/内存/WAN/在线设备） |
| 防火墙 | UFW 状态 | 防火墙 |
| Tunnel | cloudflared 运行状态 | Tunnel |
| 健康检查 | TCP 端口探测 | N150、各服务 |

**颜色语义**：
- 绿色：正常在线
- 红色：不可达/故障
- 灰色：未知/检查中

## 4. 编辑模式

### 4.1 技术选型

基于 **React + @xyflow/react**（React Flow），通过 Vite 构建为 IIFE 格式，嵌入 EJS 模板中。

- 构建输出：`public/topology-editor/editor.js` + `wemonitor.css`
- 启动方式：`window.mountTopologyEditor()` 函数
- 与主站共享：CSS 变量体系、Session 认证

### 4.2 编辑能力

- 拖拽节点模板到画布（Internet、服务器、路由器、防火墙、Tunnel、CDN、服务、ISP、设备、光猫）
- 节点间连线（边使用 `lineStyle` 属性，避免与 React `style` prop 冲突）
- 自定义节点位置、标签

### 4.3 配置持久化

- 存储：`data/topology.json`
- API：`GET /api/v1/topology-config`（读取）、`POST /api/v1/topology-config`（保存）
- 默认配置：`lib/default-topology.json`

### 4.4 查看模式 vs 编辑模式

- **查看模式**：静态 SVG 渲染 + 动态状态覆盖（页面加载时渲染）
- **编辑模式**：React Flow 交互式编辑器（按钮切换进入）

切换编辑器时，查看模式的拓扑数据作为编辑器初始状态。

## 5. API 端点

| 方法 | 端点 | 说明 |
|------|------|------|
| `GET` | `/api/v1/physical-topology` | 物理拓扑实时状态 |
| `GET` | `/api/v1/topology-config` | 读取拓扑配置 |
| `POST` | `/api/v1/topology-config` | 保存拓扑配置 |
