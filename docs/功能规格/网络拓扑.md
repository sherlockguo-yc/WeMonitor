# WeMonitor — 网络拓扑

> 状态：已实现
> 最后更新：2026-07-21

## 1. 功能概述

通过网络拓扑页面（`/network`）可视化展示 N150 家庭网络环境，包括从 Internet 到 N150 上各服务的完整链路。支持**查看模式**（静态 SVG + 动态状态）和**编辑模式**（React Flow 可视化编辑器）。

### 核心设计

- 查看模式和编辑模式共享同一份 JSON 配置（`data/topology.json`）
- 编辑器中保存 → 查看模式实时反映最新拓扑
- 编辑器中移除所有节点 → 查看模式显示空白画布

## 2. 拓扑结构

### 2.1 网络路径

**公网访问**（左列，不经过家庭网络）：
```
Internet → CF CDN → CF Tunnel → N150 → 服务
```

**局域网访问**（右列，走家庭网络）：
```
手机/电脑 → 路由器 → UFW → N150 → 服务
```

**N150 出站**（右列底部，依赖家庭网络）：
```
N150 → 路由器 → 光猫 → ISP → Internet
```

> 关键纠正：公网用户通过 Cloudflare CDN+Tunnel 访问 N150 时**不经过光猫/路由器**。光猫/路由器仅用于 N150 的主动出站（建立 CF Tunnel、系统更新等）和局域网设备联网。

### 2.2 节点类型

| 节点类型 | 说明 | 动态状态 |
|---------|------|---------|
| Internet | 互联网 | 无 |
| ISP（中国移动） | 运营商 | 无 |
| 光猫 | CMCC ONT（192.168.1.1） | Ping 检测 |
| 路由器 | 小米路由器（192.168.31.1） | Ping + API |
| 本地设备 | 手机/电脑 | 无 |
| Cloudflare CDN | CDN 节点 | 无 |
| Cloudflare Tunnel | Cloudflare 隧道 | cloudflared 进程状态 |
| UFW 防火墙 | N150 防火墙 | `ufw status` |
| N150 服务器 | 运行服务的主机 | 进程在线 + CPU/内存 |
| 服务节点 | WeMusic / WeMonitor / Webhook / WeDownload / SSH | TCP 健康检查 |

## 3. 动态状态覆盖

### 3.1 状态来源

| 状态来源 | 检测方式 | 影响节点 |
|---------|---------|---------|
| 物理拓扑 | Ping（光猫 192.168.1.1、路由器 192.168.31.1） | 光猫、路由器 |
| 路由器 API | 小米路由器 API（stok token 认证） | 路由器详情（型号/CPU/内存/WAN/在线设备） |
| 防火墙 | `ufw status verbose` | UFW |
| Tunnel | cloudflared 运行状态 | CF Tunnel、SSH |
| 健康检查 | TCP 端口探测 | N150、各服务 |

### 3.2 颜色语义

| 颜色 | 含义 | 触发条件 |
|------|------|---------|
| 绿色 | 正常在线 | 物理设备可达 / 服务 healthy / 防火墙 active |
| 橙色 | 警告 | 部分异常（如防火墙已关闭但规则存在） |
| 红色 | 异常/故障 | 不可达 / 服务 unhealthy / 隧道断开 |
| 灰色 | 未知/静态 | 无动态状态 / 数据未加载 |

### 3.3 手动颜色覆盖

编辑器支持**手动设置节点边框颜色**（双击节点 → 属性弹窗 → 颜色选择器）：
- 设置了手动颜色 → 使用该颜色（**忽略动态状态自动色**）
- 未设置（"自动（按状态）"）→ 按动态状态自动着色
- 保存后，查看模式的静态 SVG 也遵循手动颜色优先策略

## 4. 编辑模式

### 4.1 技术选型

基于 **React + @xyflow/react**（React Flow），通过 **Vite lib 模式**构建为 **IIFE** 格式，嵌入 EJS 模板中。

- 构建输出：`public/topology-editor/editor.js`（~243KB gzipped）+ `wemonitor.css`
- 启动方式：`window.mountTopologyEditor('topology-editor-root')`
- 与主站共享：CSS 变量体系、Session 认证
- Vite 配置关键参数：
  - `define: { 'process.env': '{}' }` — IIFE 模式无 process.env，React Flow 内部引用需要
  - `formats: ['iife']` — 单文件浏览器可执行格式
  - `fileName: () => 'editor.js'` — 固定文件名 + EJS `?v=<%= version %>` 破浏览器缓存

### 4.2 编辑能力

#### 节点操作

| 操作 | 方式 |
|------|------|
| 新增节点 | 从左侧面板拖入画布（10 种模板） |
| 移动节点 | 直接拖拽 |
| 编辑属性 | 双击节点 → 弹窗编辑（标签、端口、宽度、颜色） |
| 删除节点 | 选中 → 按 `Delete` |
| 连线 | 鼠标悬停节点 → 上方/下方出现蓝色圆点 → 拖拽圆点到目标节点 |

#### 连线操作

| 操作 | 方式 |
|------|------|
| 新增连线 | 从源节点 Handle 拖到目标节点 |
| 编辑标签 | 双击连线 → 弹窗编辑（标签 + 实线/虚线切换） |
| 删除连线 | 选中 → 按 `Delete` |

#### 属性编辑弹窗（PropertyModal）

双击节点/连线弹出**自定义样式弹窗**（替代浏览器原生 `prompt()`）：

- **节点编辑器**：标签（支持 `\n` 多行）、端口号、节点宽度、边框颜色（7 色板）
- **连线编辑器**：标签文本、实线/虚线切换
- 快捷键：`Enter` 保存、`Escape` 取消
- 背景遮罩点击取消

#### 节点模板

| 图标 | 类型 | 默认宽度 | 默认颜色 |
|------|------|---------|---------|
| 🌐 | Internet | 120 | 紫色 |

| 🛡️ | 防火墙 | 130 | 红色 |
| 🔗 | Tunnel | 140 | 深紫 |
| ☁️ | CDN | 120 | 粉色 |
| ⚙️ | 服务 | 120 | 青色 |
| 🏢 | ISP | 100 | 黄绿 |
| 📱 | 设备 | 120 | 灰色 |
| 🔌 | 光猫 | 100 | 蓝绿 |

### 4.3 配置持久化

```
编辑器 Save → POST /api/v1/topology-config → 写入 data/topology.json
                                            ↓
查看模式 Load → GET /api/v1/topology-config → 读取 data/topology.json → 渲染 SVG
```

- **初始化**：首次访问时，API 从 `lib/default-topology.json` 复制到 `data/topology.json`
- **data/ 目录**：受 `rsync --exclude data` 保护，不被部署覆盖
- **线程安全**：N150 单进程，无并发写问题

### 4.4 性能优化

| 优化项 | 说明 |
|-------|------|
| React.memo | `TopologyNode` 包裹 `React.memo`，data 未变化的节点跳过重渲染 |
| computeStatuses 增量 | 15 秒轮询时，只在状态实际变化时才返回新 nodes 引用 |
| 空节点兼容 | nodes 为空时，静态 SVG 使用默认 viewBox（400×300），避免 NaN |
| 定时器稳定性 | 使用 `setNodes(nds => ...)` 函数式更新，不依赖外部 state |

## 5. API 端点

| 方法 | 端点 | 认证 | 说明 |
|------|------|------|------|
| `GET` | `/api/v1/topology-config` | Session | 读取拓扑 JSON |
| `POST` | `/api/v1/topology-config` | Session | 保存拓扑 JSON |
| `GET` | `/api/v1/physical-topology` | Session | 物理拓扑实时状态 |
| `GET` | `/api/v1/firewall/status` | Session | 防火墙状态 |
| `GET` | `/api/v1/tunnel/status` | Session | 隧道状态 |
| `GET` | `/api/v1/health` | Session | 健康检查数据 |

### 5.1 数据格式

#### JSON 节点

```json
{
  "id": "n150",
  "type": "topology",
  "position": { "x": 30, "y": 310 },
  "data": {
    "label": "N150 服务器",
    "port": null,
    "dynamic": "n150",
    "healthIdx": null,
    "width": 250,
    "color": null
  }
}
```

#### JSON 边

```json
{
  "id": "e-pub3",
  "source": "cf-tunnel",
  "target": "n150",
  "label": "",
  "lineStyle": "solid"
}
```

> ⚠️ 边使用 `lineStyle`（非 `style`），避免与 React `style` prop（要求 CSS 对象）冲突。

## 7. 已知限制

- 编辑模式下无权限制约——任何已登录用户都能修改拓扑（服务只供可信用户使用）
- 无版本历史/撤销——每次保存覆盖前次配置
- 静态视图不支持虚线渲染——所有边统一实线
