# WeMonitor — 网络拓扑

> 状态：已实现
> 最后更新：2026-07-22

## 1. 功能概述

通过网络拓扑页面（`/network`）可视化展示 N150 家庭网络环境，包括从 Internet 到 N150 上各服务的完整链路。

**统一画布**：使用 React Flow 单一画布，编辑器和查看器是同一个实例。默认**只读模式**（可缩放/平移/悬停查看），点击「编辑」解锁拖拽/连线/属性修改。

> 不再有独立的静态 SVG 渲染——编辑画布 = 查看画布，避免两套渲染对齐不一致。

## 2. 模式切换

| 模式 | 行为 |
|------|------|
| **只读**（默认） | 悬停节点查看 tooltip（名称+状态+端口）· 滚轮缩放 · 拖拽平移画布 · 禁用拖拽/连线/删除/双击编辑 |
| **编辑**（点击按钮） | 左侧面板拖入节点 · 节点 Handle 拖线连接 · 双击改属性 · Delete 删除 · 保存按钮出现 |

## 3. 拓扑结构

### 3.1 网络路径

**公网访问**（左列，不经过家庭网络）：
```
Internet → CF CDN → CF Tunnel → N150 → 服务
```

**局域网访问**（右列，走家庭网络）：
```
手机/电脑 → 路由器 → UFW → N150 → 服务
```

**N150 出站**（依赖家庭网络）：
```
N150 → 路由器 → 光猫 → ISP → Internet
```

### 3.2 节点类型

| 节点类型 | 说明 | 动态状态 |
|---------|------|---------|
| Internet | 互联网 | 无 |
| ISP（中国移动） | 运营商 | 无 |
| 光猫 | CMCC ONT（192.168.1.1） | Ping 检测 |
| 路由器 | 小米路由器（192.168.31.1） | Ping + API |
| 本地设备 | 手机/电脑 | 无 |
| Cloudflare CDN | CDN 节点 | 无 |
| Cloudflare Tunnel | Cloudflare 隧道 | cloudflared 进程状态 |
| UFW 防火墙 | N150 防火墙 | `ufw status` |
| N150 服务器 | 运行服务的主机 | 进程在线 + CPU/内存 |
| 服务节点 | WeMusic / WeMonitor / Webhook / WeDownload / SSH | TCP 健康检查 |

## 5. 动态状态

### 状态来源

| 来源 | 方式 | 影响节点 |
|------|------|---------|
| 物理拓扑 | Ping | 光猫、路由器、N150 |
| 小米路由器 API | stok 认证 HTTP | 路由器详情 |
| 防火墙 | `ufw status verbose` | UFW |
| Tunnel | cloudflared 进程 | CF Tunnel、SSH |
| 健康检查 | TCP 探测 | N150、各服务 |

### 颜色规则

**优先级**：手动覆盖色 > 动态状态自动色。

| 颜色 | 含义 |
|------|------|
| 绿 | 在线/healthy/active |
| 橙 | 警告 |
| 红 | 异常/不可达 |
| 灰 | 未知/无动态状态 |

## 7. 版本历史

每次点击「保存」，系统会在写入 `topology.json` 之前自动创建版本快照。

### 存储

```
data/topology.json              ← 当前版本
data/topology-versions/
├── v_2026-07-22T10-30-00.json  ← 历史快照
├── v_2026-07-21T15-20-00.json
└── ...
```

- 每个版本是完整的 `topology.json` 副本
- 如果内容和上一个版本完全相同则跳过（去重）
- 最多保留 50 个版本，超出自动删除最旧的

### 操作

| 操作 | 方式 |
|------|------|
| 查看版本列表 | 工具栏「版本历史」→ 弹窗显示时间、节点数、连线数 |
| 预览版本内容 | 点击「预览」→ 展开节点/连线列表 |
| 恢复版本 | 点击「恢复」→ 二次确认 → 恢复前自动保存当前版本为快照 |

恢复后编辑器自动刷新加载恢复后的数据。

## 8. 配置持久化

```
编辑器 Save → POST /api/v1/topology-config → 写入 data/topology.json
页面加载 → GET /api/v1/topology-config → 读取 → React Flow 渲染
```

- **初始化**：首次 `GET` 时从 `lib/default-topology.json` 复制到 `data/topology.json`
- **data/ 目录**：`rsync --exclude data` 保护，不被部署覆盖

### JSON 格式

**节点**：`{ id, type:"topology", position:{x,y}, data:{label, port, dynamic, color, width} }`
**边缘**：`{ id, source, target, label, lineStyle, edgeType, arrow }`（`lineStyle` 替代 `style` 避免 React prop 冲突；`arrow` 默认 `true`，兼容旧数据）

## 9. API

| 方法 | 端点 | 说明 |
|------|------|------|
| `GET` | `/api/v1/topology-config` | 读取拓扑 JSON |
| `POST` | `/api/v1/topology-config` | 保存拓扑 JSON（自动创建版本快照） |
| `GET` | `/api/v1/topology-config/versions` | 版本列表 |
| `GET` | `/api/v1/topology-config/versions/:id` | 获取指定版本完整数据 |
| `POST` | `/api/v1/topology-config/versions/:id/restore` | 恢复指定版本（恢复前自动保存当前版本） |
| `GET` | `/api/v1/physical-topology` | 物理状态 |
| `GET` | `/api/v1/firewall/status` | 防火墙状态 |
| `GET` | `/api/v1/tunnel/status` | 隧道状态 |
| `GET` | `/api/v1/health` | 健康检查 |

## 11. 已知限制

- 编辑模式无权限约束（服务供可信用户使用）
