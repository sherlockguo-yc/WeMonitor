# WeMonitor — 定时任务管理

> 状态：已实现

## 1. 功能概述

通过 Web 界面管理 N150 服务器的 crontab 定时任务，支持可视化创建、编辑、启停、查看运行历史，无需 SSH 到服务器手动编辑 crontab。

## 2. 核心设计

### 2.1 非侵入式 crontab 管理

WeMonitor 使用标记块区分自己管理和用户手动添加的 crontab 条目：

```
# === WeMonitor BEGIN ===
*/5 * * * * /path/to/job1.sh
0 * * * * /path/to/job2.sh
# === WeMonitor END ===
```

- BEGIN/END 标记块之外的内容完全保留不动
- 用户手动添加的 cron 条目与 WeMonitor 管理的条目和平共存

### 2.2 持久化

- 任务配置：`data/cron-jobs.json`（JSON 文件）
- 运行历史：`data/cron-history.jsonl`（JSONL 格式，保留最近 1000 条）

### 2.3 执行器

`scripts/cron-runner.js` — crontab 中每个 WeMonitor 任务调用此脚本执行：
1. 通过 `child_process.spawn` 执行实际命令
2. 捕获 stdout/stderr
3. 记录 exit code、开始/结束时间
4. 追加到 `cron-history.jsonl`

### 2.4 同步机制

每次 CRUD 操作自动调用 `syncToSystem()` 将最新任务列表写入系统 crontab。

**手动同步**：`POST /api/v1/cron/sync` — 强制将内存中的任务列表同步到系统 crontab。

**同步状态**：`GET /api/v1/cron/sync-status` — 检查系统 crontab 与本地 JSON 是否一致，不一致时前端显示警告。

### 2.5 首次导入

系统启动时从 crontab 的 WeMonitor 标记块中解析已有任务，导入到 `cron-jobs.json`。如果有用户手动在标记块内添加了条目，它们会被自动纳入 WeMonitor 管理。

## 3. 任务配置

| 字段 | 说明 |
|------|------|
| `name` | 任务名称 |
| `schedule` | Cron 表达式（5 字段：分/时/日/月/周） |
| `command` | 要执行的命令 |
| `enabled` | 是否启用 |

**表达式校验**：前后端双重校验 cron 表达式合法性（5 字段格式）。

## 4. 前端页面 (`/cron`)

### 4.1 任务列表

- 表格展示所有任务：名称、调度表达式、命令、状态
- Toggle 开关启停任务
- 同步状态警告横幅（crontab 与 JSON 不一致时显示）

### 4.2 新增/编辑模态框

- 表单：名称、cron 表达式（5 字段）、命令
- **预设模板**：常见调度（每分钟/每 5 分钟/每小时/每天/每周）一键填充
- **人类可读预览**：输入 cron 表达式后实时显示自然语言描述

### 4.3 运行历史

点击任务 → 弹窗显示最近执行记录：
- 开始/结束时间
- Exit code
- stdout/stderr 输出

## 5. API 端点

| 方法 | 端点 | 说明 |
|------|------|------|
| `GET` | `/api/v1/cron/jobs` | 任务列表 |
| `POST` | `/api/v1/cron/jobs` | 创建任务 |
| `PUT` | `/api/v1/cron/jobs/:id` | 更新任务 |
| `DELETE` | `/api/v1/cron/jobs/:id` | 删除任务 |
| `POST` | `/api/v1/cron/jobs/:id/toggle` | 启停任务 |
| `GET` | `/api/v1/cron/jobs/:id/history` | 运行历史 |
| `POST` | `/api/v1/cron/sync` | 强制同步 crontab |
| `GET` | `/api/v1/cron/sync-status` | 同步状态 |

## 6. 关键文件

| 文件 | 用途 |
|------|------|
| `lib/cron.js` | 定时任务管理器（CRUD/同步/首次导入） |

| `data/cron-jobs.json` | 持久化任务配置 |
| `data/cron-history.jsonl` | 运行历史 |
