# WeMonitor — 系统资源监控

> 状态：已实现

## 1. 功能概述

自动采集 N150 服务器的系统资源指标（CPU、内存、磁盘、网络、温度、负载、运行时间），以 30 秒间隔持续写入 SQLite，通过 Web Dashboard 的「概览」和「系统资源」页面展示实时数据与历史趋势图。

## 2. 采集指标

| 类别 | 指标 | 采集频率 | 数据源 |
|------|------|---------|--------|
| CPU | `cpu_usage_percent` | 30s | `systeminformation` |
| 内存 | `mem_total_gb`, `mem_used_gb`, `mem_usage_percent` | 30s | `systeminformation` |
| 磁盘 | `disk_total_gb`, `disk_used_gb`, `disk_usage_percent`（按挂载点分组） | 60s | `systeminformation` |
| 网络 | `net_rx_bytes_sec`, `net_tx_bytes_sec` | 60s | `systeminformation` |
| 负载 | `load_1m` | 60s | `os.loadavg()` |
| 温度 | `cpu_temp_celsius` | 60s | `systeminformation` |
| 运行时间 | `uptime_seconds` | 60s | `os.uptime()` |

## 3. 技术实现

### 3.1 采集器

`lib/collector/system.js` — 定时任务，调用 `systeminformation` 库获取各项指标，批量写入 SQLite `metrics` 表。

### 3.2 当前快照

`GET /api/v1/stats/current` — 查询各指标最新一条记录，返回当前 CPU/内存/磁盘/网络/温度/负载/运行时间的实时值。前端「概览」页每 15 秒刷新一次。

### 3.3 历史趋势

`GET /api/v1/metrics/batch` — 批量查询多个指标的历史数据，支持按时间范围过滤。

**智能路由**：查询范围 > 1 小时 → 走 `metrics_hourly` 聚合表（性能优化）；否则走 `metrics` 原始表。

### 3.4 可视化

系统资源页使用 **uPlot** 绘制 4 个时间序列折线图：
- CPU 使用率（%）
- 内存使用量（GB）
- 磁盘使用率（%，各分区）
- 网络流量（KB/s，入站/出站）

支持 4 个时间范围 Tab：1 小时 / 6 小时 / 24 小时 / 7 天。

图表使用 `createTimeChart()` 封装（`public/js/common.js`），自动处理时间戳、Y 轴自适应、tooltip。

### 3.5 时间轴正确性

X 轴使用完整的时间范围刻度（而非按数据点数量），确保选 6h 时显示 6 个整点刻度，即使数据稀疏也不会失真。

## 4. 概览页

`GET /` — 4 个统计卡片展示实时值 + 服务健康状态列表：

| 卡片 | 内容 |
|------|------|
| CPU | 使用率百分比 |
| 内存 | 已用/总量 |
| 磁盘 | 各分区使用率 |
| 网络 | 入站/出站速率 |
