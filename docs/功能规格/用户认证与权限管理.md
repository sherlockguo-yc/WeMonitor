# WeMonitor — 用户认证与权限管理

> 状态：已实现

## 1. 功能概述

WeMonitor 提供完整的用户认证和权限体系，支持注册审批流和基于角色的页面访问控制。

## 2. 认证流程

```
注册 → 首个用户自动 admin + active
注册 → 后续用户默认 user + pending → admin 在「用户管理」审批 → active
```

## 3. 技术实现

### 3.1 密码存储

- bcrypt 加密，10 rounds
- 密码哈希存储在 SQLite `users` 表，API 不返回密码字段

### 3.2 Session 管理

- 基于 `express-session` + `better-sqlite3-session-store`
- Session 持久化到 SQLite，服务重启不影响登录状态
- 有效期 7 天，httpOnly cookie
- 中间件自动将 `session.user` 注入所有 EJS 模板

### 3.3 鉴权中间件

| 中间件 | 作用 | 未通过行为 |
|--------|------|-----------|
| `requireAuth` | 要求登录 | 页面请求 → 重定向 `/login`；API 请求 → 401 |
| `requireActive` | 要求已激活 | 重定向 `/pending` |
| `requireAdmin` | 要求 admin 角色 | 403 |

### 3.4 角色与权限

| 角色 | 状态 | 权限 |
|------|------|------|
| `admin` | active | 全部页面 + 用户管理（审批/改角色/删除） |
| `user` | pending | 仅看到「等待审批」页面 |
| `user` | active | Dashboard 全部查看权限（概览/系统/服务/防火墙/Tunnel/CI-CD/网络/定时任务/备份） |

### 3.5 管理员保护

- 不能删除/降级最后一个 admin
- 不能修改自己的角色

## 4. 数据模型

```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  username TEXT NOT NULL UNIQUE,
  password_hash TEXT NOT NULL,
  role TEXT NOT NULL DEFAULT 'user',   -- admin / user
  status TEXT NOT NULL DEFAULT 'pending'  -- active / pending
);
```

## 5. API 端点

| 方法 | 端点 | 说明 | 权限 |
|------|------|------|------|
| `GET` | `/login` | 登录页面 | 公开 |
| `POST` | `/login` | 登录处理 | 公开 |
| `GET` | `/register` | 注册页面 | 公开 |
| `POST` | `/register` | 注册处理 | 公开 |
| `GET` | `/logout` | 退出登录 | 公开 |
| `GET` | `/pending` | 等待审批页面 | 需登录 |
| `GET` | `/api/v1/admin/users` | 用户列表 | admin |
| `POST` | `/api/v1/admin/users/:id/approve` | 审批用户 | admin |
| `PUT` | `/api/v1/admin/users/:id/role` | 修改角色 | admin |
| `DELETE` | `/api/v1/admin/users/:id` | 删除用户 | admin |
