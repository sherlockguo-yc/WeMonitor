# WeMonitor — 服务健康检查

> 状态：已实现

## 1. 功能概述

对被监控服务执行 TCP 端口探测，记录健康状态和延迟，在「服务状态」页面集中展示。支持通过「服务管理」页面对服务进行 CRUD 操作和启停控制。

## 2. 健康检查机制

### 2.1 探测方式

- 类型：TCP 端口连接（`lib/collector/health.js`）
- 频率：由服务配置的 `scrape_interval` 决定（默认 30 秒）
- 判定：端口可达 → `healthy`；连接超时 → `unhealthy`；异常 → `unknown`
- 延迟记录：每次探测记录 `latency_ms`

### 2.2 检查对象

从 `services` 表中读取所有 `enabled = 1` 且有 `health_target` 的服务，逐个探测。

## 3. 服务管理

### 3.1 服务配置

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | string | 服务名称（唯一） |
| `scrape_url` | string\|null | Pull 模式的 Prometheus metrics URL，无则为 null |
| `scrape_interval` | int | 采集/检查间隔（秒） |
| `health_type` | string | 健康检查类型（目前仅 `tcp`） |
| `health_target` | string\|null | 探测目标，如 `127.0.0.1:5174` |
| `enabled` | bool | 是否启用监控 |

### 3.2 CRUD 操作

| 操作 | UI 方式 | API |
|------|--------|-----|
| 新增 | 模态框表单 | `POST /api/v1/services` |
| 编辑 | 表格行编辑 | `PUT /api/v1/services/:id` |
| 启用/禁用 | Toggle 开关 | `PATCH /api/v1/services/:id/toggle` |
| 删除 | 删除按钮 | `DELETE /api/v1/services/:id` |

### 3.3 默认服务

`config.js` 中 `defaultServices` 数组定义新安装时的默认服务（如 WeMusic、WeDownload）。首次运行时通过 `INSERT OR IGNORE` 注入数据库。

## 4. 数据模型

```sql
CREATE TABLE services (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE,
  scrape_url TEXT,
  scrape_interval INTEGER DEFAULT 30,
  health_type TEXT,
  health_target TEXT,
  enabled INTEGER DEFAULT 1
);

CREATE TABLE health_checks (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  service_id INTEGER REFERENCES services(id),
  status TEXT NOT NULL,        -- healthy / unhealthy / unknown
  latency_ms INTEGER,
  message TEXT,
  timestamp INTEGER NOT NULL
);
```

## 5. API 端点

| 方法 | 端点 | 说明 |
|------|------|------|
| `GET` | `/api/v1/health` | 各服务最新健康状态 |
| `GET` | `/api/v1/services` | 服务列表 |
| `POST` | `/api/v1/services` | 创建服务 |
| `PUT` | `/api/v1/services/:id` | 更新服务 |
| `PATCH` | `/api/v1/services/:id/toggle` | 启停服务 |
| `DELETE` | `/api/v1/services/:id` | 删除服务 |

## 6. 前端页面

### 服务状态 (`/services`)

- 服务列表表格：名称、状态点（绿/红/灰）、延迟、上次检查时间
- 30 秒自动刷新

### 服务管理 (`/settings`)

- 服务 CRUD 表格 + Toggle 开关
- 新增/编辑模态框，风格与系统其他模态框一致
